1. Бинарная куча (Binary Heap)
Что это?
Бинарная куча - это полное бинарное дерево, которое удовлетворяет свойству кучи: значение каждого родительского узла либо больше/равно (max-heap), либо меньше/равно (min-heap) значений его дочерних узлов.

Основные свойства:
Полное бинарное дерево: все уровни заполнены, кроме последнего, который заполняется слева направо

Свойство кучи: для min-heap родитель ≤ дети, для max-heap родитель ≥ дети

Эффективная реализация через массив: для узла с индексом i:

Левый ребенок: 2i + 1

Правый ребенок: 2i + 2

Родитель: ⌊(i-1)/2⌋

Операции и сложность:
Вставка: O(log n) - добавляем в конец и "просеиваем вверх"

Удаление корня: O(log n) - заменяем корень последним элементом и "просеиваем вниз"

Поиск минимума/максимума: O(1) - всегда в корне

Объединение куч: O(n) - неэффективно

Применение:
Пирамидальная сортировка (Heapsort)

Очередь с приоритетом

Алгоритм Дейкстры

2. Биномиальная куча (Binomial Heap)
Что это?
Биномиальная куча - это набор биномиальных деревьев, которые удовлетворяют свойству кучи. Каждое биномиальное дерество Bₖ имеет 2ᵏ узлов и определенную структуру.

Биномиальные деревья Bₖ:
B₀: Один узел

Bₖ: Состоит из двух Bₖ₋₁, где корень одного становится левым дочерним узлом корня другого

Свойства:
Состоит из набора биномиальных деревьев разного размера

Каждое дерево удовлетворяет свойству min-heap

Деревья упорядочены по степени (размеру)

Операции и сложность:
Вставка: O(1) амортизированная

Удаление минимума: O(log n)

Объединение: O(log n)

Поиск минимума: O(log n) - нужно проверить корни всех деревьев

Преимущества перед бинарной кучей:
Эффективное объединение куч

Амортизированно быстрая вставка

3. Куча Фибоначчи (Fibonacci Heap)
Что это?
Куча Фибоначчи - это коллекция деревьев, удовлетворяющих свойству min-heap. Оптимизирована для операций уменьшения ключа и объединения.

Особенности структуры:
Набор деревьев (не обязательно биномиальных)

Корни деревьев связаны в циклический двусвязный список

Поддержание "ленивого" подхода к объединению деревьев

Ключевые концепции:
Marked nodes: помеченные узлы, которые потеряли дочерний узел

Consolidation: процесс объединения деревьев одинакового размера

Операции и сложность (амортизированная):
Вставка: O(1)

Удаление минимума: O(log n)

Поиск минимума: O(1)

Уменьшение ключа: O(1)

Объединение: O(1)

Применение:
Алгоритм Дейкстры (ускоренная версия)

Алгоритм Прима для MST

Когда много операций уменьшения ключа

4. Хеш-таблицы (Hash Tables)
Что это?
Хеш-таблица - это структура данных, которая реализует ассоциативный массив, отображающий ключи в значения с помощью хеш-функции.

Основные компоненты:
1. Хеш-функция:
Преобразует ключ в индекс массива

Должна быть детерминированной и равномерно распределять ключи

Пример: hash(key) = key % table_size

2. Разрешение коллизий:
Метод цепочек: каждый бакет содержит список элементов

Открытая адресация: поиск следующего свободного слота

Операции и сложность (в среднем):
Вставка: O(1)

Поиск: O(1)

Удаление: O(1)

Ключевые параметры:
Load Factor = n/k (n - элементы, k - бакеты)

Обычно resize при load factor ≈ 0.75

Применение:
Словари, множества

Кэширование

Базы данных

Когда что использовать?
Бинарная куча: простые случаи, когда не нужно частое объединение

Биномиальная куча: когда нужны частые объединения куч

Куча Фибоначчи: алгоритмы с множественными операциями уменьшения ключа

Хеш-таблица: быстрый поиск по ключу, ассоциативные массивы
